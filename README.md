## Git Commit Guidelines

コミットメッセージのプレフィックス例

|  prefix  | 内容 |
|----------|------|
| feat     |  新機能 |
| fix      |  バグの修正 |
| docs     |  ドキュメント修正 |
| style    |  コード内容に影響を与えない変更 <br> 例. white-space, formatting, missing semi-colons |
| refactor |  リファクタリング<br> バグの修正も機能の追加も行わないコード変更 |
| perf     |  パーフォマンスを向上させるための変更 |
| test     |  テストの追加・修正 |
| chore    |  雑用<br>ビルドに関する変更やドキュメント生成のような変更 |
| wip      |  作業中 |
